ni (Next Instruction): 한 줄씩 어셈블리 명령어를 실행합니다.
c (Continue): 다음 브레이크포인트까지 계속 실행합니다.

추가 내용
1. 명령어 구조: x/[개수][형식][크기] [주소]
[개수] (N): 몇 개나 보여줄 것인가? (예: x/4x는 4개를 보여줌)

[형식] (Op): 데이터를 어떤 형태로 해석해서 보여줄 것인가?

[크기]: 한 단위의 크기 (b=byte, h=halfword, w=word, g=giant/8bytes)

2. 형식(Format) 옵션 상세
질문하신 Op 부분에 들어갈 옵션들은 데이터를 어떻게 해석할지 결정합니다.

x (Hexadecimal): 16진수(Hex) 형태로 출력합니다. 메모리 값을 있는 그대로 보고 싶을 때 가장 많이 씁니다.

i (Instruction): 어셈블리어(Disassembly) 형태로 해석합니다. CPU가 이 메모리 위치에 있는 데이터를 '명령어'로 이해하도록 출력합니다.

s (String): 문자열(String)로 해석합니다. 해당 주소부터 NULL 문자(0x00)를 만날 때까지를 문자로 간주하여 화면에 출력합니다. (메모리에 저장된 텍스트를 읽을 때 필수!)
